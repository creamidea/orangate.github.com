Windows 8的体验与感想

Nov 5th, 2012

近日给两台笔记本电脑安装了Windows 8，感受颇多

微软在发布正式版时声称win8已经经历1.4亿小时的测试，可见这次微软在开发测试上是下了血本

Win8与前几代操作系统操作上有显著差异。Metro界面明显更适合带触摸屏的设备，开始菜单也取消了。要熟练使用win8需要记住几个常用快捷键，比如win+c和win+q之类。当然，也可以在Metro界面点击”桌面”回到我们熟悉的类似于win7的界面

Win8优化得很不错，运行Win7很费劲的极低端CPU(比如AMD C-50 1Ghz主频）运行Win8都相当流畅，而且开了很多的后台进程。各个界面的切换毫无延迟，动画效果相当出彩。至于内存，2GB绰绰有余了。简而言之，win8对硬件配置要求很低

Win8的一大亮点就是应用商店，这类似于Itunes或Google Play，绑定Live Mail帐号，可以下载安装软件和购买软件。目前应用商店里面软件不多，但增速惊人

应用商店里的软件要通过微软审核，在人家的地盘当然不能放肆。Win8版QQ仅有7MB，人人仅有2MB，与移动应用差不多大小，没有乱七八糟的广告，界面为Metro风格，启动速度极快，令人满意。希望这是个规范各种软件的好开端！

Win8目前最大的问题是驱动，毕竟正式版还没有发布多久，各个硬件厂商还没有普及win8的驱动。在安装时，系统会自动检测硬件，对于大部分集成的硬件，驱动兼容得很好。但是对于高端的显卡声卡，或者过于陈旧的显卡，驱动显然还要等厂商来做

我测试的两台电脑中，一台电脑的ATI Radeon HD2XXX（很古老的低端显卡）显卡驱动不正常，只能以兼容模式运行，画面发虚。另一台ATI Radeon HD6250官方已经有Win8驱动，显示效果非常优秀。但是声卡驱动不正常，大量出现破音爆音

另一个非常让我们关注的问题是Windows 8的破解问题。简而言之，Windows 8的完美破解将非常困难

Windows 7在OEM认证上，微软留下了巨大的漏洞：通过开机验证主板上的SILC版本，以及系统中证书和密匙的匹配，来验证是否为OEM版本。大家很轻易地通过修改BIOS而把主板修改为了SILC 2.1（虽然存在风险），从而非常完美地通过了激活。

Windows 8的OEM认证方式与Win7不同，目前没有破解。目前主要激活方式是通过自己架设KMS服务器实现180天激活。

Windows 8最为重要的意义，在于建立一个大一统的格局，这也是微软的野心—千秋万代，一统江湖。

Windows 8的X86版本，ARM版本以及Windows Phone 8的高度兼容，使得多平台的软件可以互相移植。Windows Phone 8手机与windows 8的协同性将前所未有–不光是文档，音乐，outlook等的同步，还有应用软件的同步。靠垄断而获取暴利的Apple IOS，靠开源而大量占有市场的Google Andriod，恐怕在不久的将来都要靠边站……

Windows 8预计将在半年内获得全面的驱动支持，各种漏洞也基本解决。到那时，我的电脑将全面由Windows 7升级Windows 8
